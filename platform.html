How It Works

Why Containers Win

Containers package apps and dependencies and start far faster than full VMs, so users hit Launch and get to work in seconds.

Container-Based Creation

Container tasks avoid VM provisioning delays and benefit from image caching for even faster subsequent starts. Scale elastically across clusters or regions; run dozens of secured workspaces per node where VMs would run only a few.
